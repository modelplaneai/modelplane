Install

pip install modelplane-cli

Requires Python 3.9+ and kubectl configured with access to a ModelPlane cluster.

Quick start

# Set your team (one-time)
$ mp init --team ml-team
Team set to: ml-team

# See what models are available
$ mp models

# Deploy one
$ mp deploy qwen-0.5b-vllm --env prod-gpu-east

# Wait for it
$ mp status qwen-0.5b-vllm --watch

# Test it
$ mp predict qwen-0.5b-vllm -i "Explain attention in transformers"

mp deploy

Deploy a model from the catalog or from a YAML file.

From catalog:    mp deploy llama3-8b [--env prod-gpu-east]
From YAML file:  mp deploy -f model.yaml [--env prod-gpu-east]
Autoscale:       mp deploy llama3-8b --min 1 --max 6 --target 32
Scale to zero:   mp deploy llama3-8b --scale-to-zero --max 4 --target 16
mp deploy [<model>] [options]

Options

FlagDescriptionDefault
-f, --file PATHPath to a Model YAML file.
--env TEXTTarget a specific InferenceEnvironment by name.
--envs INTEGERNumber of environments to fan out across.1
--name TEXTDeployment name (defaults to model name).
--team TEXTOverride team.
--replicas INTEGERFixed pod count per placement (signal: Fixed). Default 1.
--min INTEGERMin replicas for autoscaling (signal: Concurrency).
--max INTEGERMax replicas for autoscaling (signal: Concurrency).
--target INTEGERTarget in-flight requests per replica (signal: Concurrency).
--scale-to-zeroShorthand for --min 0 (still requires --max and --target).
--utilization INTEGERScale at this percent of --target. Default 70.
--scale-down-delay INTEGERSeconds before removing replicas. Default 300.

Agents and scripting

The CLI is designed for AI agents and shell scripts as first-class callers, not just humans at a terminal. The properties below are stable across the v0.1.x series.

Exit codes

CodeMeaning
0Success
1Generic error (network, unexpected exception)
2Usage error (bad flags, missing required argument, non-TTY without -y)
3Resource not found (deployment, model, environment)
4Backend or cluster error (CRD validation failed, kubectl error, model unhealthy)
5Timeout (e.g. --watch exceeded its deadline)

TTY-aware behavior

  • Confirmation prompts (mp delete) only render when stdout is a TTY. Off-TTY: pass -y or the command exits 2.
  • Progress animations (mp status --watch) only render on a TTY. Off-TTY: one line per state transition.
  • Color is auto-detected; force with --color always or disable with NO_COLOR=1.

Deterministic startup

No telemetry. No version checks on startup. No auto-updates. Every invocation is a function of its arguments, environment, and cluster state — safe to run inside agents and CI without surprise network calls.

Self-describing

mp --help and mp <command> --help are the canonical command reference. This page is generated from the same Click command tree, so anything --help shows is also documented here.

Configuration

Team context

The CLI stores your team name in ~/.config/modelplane/config.yaml. This maps to a Kubernetes namespace. Resolution order:

  1. --team flag on any command
  2. MP_TEAM environment variable
  3. Saved config from mp init --team
  4. Falls back to default

Cluster access

The CLI uses your existing kubeconfig — the same config kubectl uses. If kubectl get clustermodels works, the CLI will work.

Environment variables

VariableEffect
MP_TEAMOverride the team for all commands
NO_COLORDisable colored output
KUBECONFIGStandard kubectl config path; honored as-is
