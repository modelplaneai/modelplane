
# Modelplane separates two roles. The PLATFORM TEAM builds a GPU fleet with published hardware capabilities. The ML TEAM deploys models against those capabilities — by capability, never by cluster name.

# The control plane is already running on kind: Crossplane, the Modelplane package, and GCP credentials.

# Part 1 (first-deployment.md) — stand up the platform and serve one model.

# PLATFORM TEAM installs the control-plane routing gateway. It runs Traefik (+ MetalLB on kind) so every ModelService gets one stable address:
$ cat ../../qwen-demo/01-gateway.yaml
# The control plane routing gateway. Installs Traefik Proxy and MetalLB
# (for kind clusters). On cloud clusters, omit the loadBalancer/metallb
# fields — the cloud LB controller handles address assignment.
apiVersion: modelplane.ai/v1alpha1
kind: InferenceGateway
metadata:
  name: default
spec:
  backend: Traefik
  traefik:
    version: "40.2.0"
    loadBalancer: MetalLB
    metallb:
      addressPool: "172.18.255.200-172.18.255.250"

# ML TEAM declares a model and one endpoint — the selector asks for GPU memory (>= 20Gi), never a cluster name:
$ manifest 01-first-deployment.yaml 'ModelDeployment|ModelService'
apiVersion: modelplane.ai/v1alpha1
kind: ModelDeployment
metadata:
  name: qwen-demo
  namespace: ml-team
spec:
  replicas: 1
  engines:
  - name: qwen
    members:
    - role: Standalone
      nodeSelector:
        devices:
        - name: gpu
          count: 1
          selectors:
          - cel: |
              device.capacity["gpu.nvidia.com"].memory.compareTo(quantity("20Gi")) >= 0
      template:
        spec:
          containers:
          - name: engine
            image: vllm/vllm-openai:v0.11.0
            args:
            - --model=Qwen/Qwen2.5-0.5B-Instruct
            - --dtype=half
---
apiVersion: modelplane.ai/v1alpha1
kind: ModelService
metadata:
  name: qwen
  namespace: ml-team
spec:
  endpoints:
  - selector:
      matchLabels:
        modelplane.ai/deployment: qwen-demo
---

# Part 2 (scale-to-fleet.md) — the model needs more GPU. The platform grows; the ML team edits the same deployment. No model swap, no cluster names.

# PLATFORM TEAM adds a bigger hardware class (A100 40GB) and two clusters that offer it, in new regions:
